Preguntas

  1. ¿Qué es el DOM?

    El DOM (Document Object Model) es una representación estructurada en forma de árbol de un documento HTML o XML que permite a los programas acceder y manipular el contenido, estructura y estilos de una página web. Actúa como una interfaz de programación que conecta las páginas web con scripts o lenguajes de programación.

    
// Ejemplo de acceso al DOM
document.getElementById("ejemplo").innerHTML = "Nuevo contenido";

    Fuente: MDN Web Docs - Introducción al DOM

    Según la especificación oficial del W3C, el DOM no solamente define la estructura de los documentos, sino también el comportamiento de éstos y una forma estándar de acceso que permite que lenguajes de programación (no solo JavaScript) puedan interactuar con el documento de manera consistente en diferentes plataformas y navegadores.

    Fuente adicional: W3C DOM Standard

  2. ¿Qué es un nodo?

    Un nodo es cualquier objeto individual dentro del árbol del DOM. Existen varios tipos de nodos, siendo los más comunes: elementos (como <div>, <p>), atributos, texto, comentarios y el documento mismo. Cada nodo puede tener nodos padres, hermanos y/o hijos, formando así la estructura jerárquica del DOM.

    
// Ejemplo de navegación entre nodos
let padre = documento.body;
let primerHijo = padre.firstChild;
let siguienteHermano = primerHijo.nextSibling;

    Fuente: MDN Web Docs - Node

    Según JavaScript.info, en el DOM hay 12 tipos de nodos, pero en la práctica generalmente trabajamos con los 4 principales: Document (nodo raíz), Element (etiquetas HTML), Text (contenido textual) y Comment (comentarios). Cada tipo de nodo tiene propiedades diferentes y una adecuada manipulación de ellos permite optimizar el rendimiento de las aplicaciones web.

    Fuente adicional: JavaScript.info - DOM Nodes

  3. ¿Cómo se accede a un elemento del DOM?

    Se puede acceder a elementos del DOM mediante varios métodos, entre los más utilizados están:

    
// Por ID
const elemento = document.getElementById('miId');

// Por clase (devuelve una colección)
const elementos = document.getElementsByClassName('miClase');

// Por etiqueta (devuelve una colección)
const parrafos = document.getElementsByTagName('p');

// Usando selectores CSS (primer coincidencia)
const primerElemento = document.querySelector('.miClase');

// Usando selectores CSS (todas las coincidencias)
const todosLosElementos = document.querySelectorAll('div.miClase');

    Fuente: MDN Web Docs - Métodos de selección

  4. ¿Cómo se modifica un elemento del DOM?

    Los elementos del DOM pueden modificarse de varias maneras:

    
// Modificar contenido
elemento.textContent = "Nuevo texto";  // Solo texto
elemento.innerHTML = "Texto con HTML";  // Incluye HTML

// Modificar atributos
elemento.setAttribute("class", "nueva-clase");
elemento.id = "nuevoId";

// Modificar estilos
elemento.style.color = "red";
elemento.style.backgroundColor = "#f0f0f0";

// Agregar/eliminar clases
elemento.classList.add("nueva-clase");
elemento.classList.remove("vieja-clase");
elemento.classList.toggle("clase-toggle");

// Crear y añadir elementos
const nuevoElemento = document.createElement("div");
nuevoElemento.textContent = "Soy nuevo";
elemento.appendChild(nuevoElemento);

// Eliminar elementos
elemento.removeChild(elemento.childNodes[0]);
// o
elementoAEliminar.parentNode.removeChild(elementoAEliminar);

    Fuente: MDN Web Docs - Element

    Según Eloquent JavaScript, cuando se modifican elementos del DOM, es importante considerar el impacto en el rendimiento de la página. Las operaciones que causan un recálculo del diseño (reflow) son costosas, especialmente en bucles. Una técnica óptima es utilizar fragmentos de documentos para construir estructuras complejas fuera del DOM y luego insertarlas de una sola vez:

    
// Uso de DocumentFragment para operaciones por lotes
const fragment = document.createDocumentFragment();
for (let i = 0; i < 1000; i++) {
    const newElement = document.createElement("li");
    newElement.textContent = `Elemento ${i}`;
    fragment.appendChild(newElement);
}
// Solo un reflow ocurre con esta única operación
document.getElementById("lista").appendChild(fragment);

    Fuente adicional: Eloquent JavaScript - Manipulación del DOM

  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    Las principales diferencias son:

    • Selectores: getElementBy* usa identificadores específicos (ID, clase, etiqueta), mientras que querySelector* usa selectores CSS más flexibles.
    • Retorno: getElementBy* devuelve colecciones HTML vivas (que se actualizan automáticamente), mientras que querySelector* devuelve NodeList estáticas.
    • Rendimiento: getElementBy* suele ser más rápido, pero querySelector* ofrece mayor flexibilidad.
    • Compatibilidad: getElementBy* tiene mayor compatibilidad con navegadores antiguos.
    
// getElementBy* - devuelve colección viva
const divs = document.getElementsByTagName('div');
// Si se añade un nuevo div, 'divs' se actualiza automáticamente

// querySelector* - devuelve NodeList estática
const divs2 = document.querySelectorAll('div');
// Si se añade un nuevo div, 'divs2' NO se actualiza

    Fuente: MDN Web Docs - Diferencias en métodos de selección

    Según CSS-Tricks, aunque getElementBy* puede ser ligeramente más rápido en algunos casos, querySelector* ofrece una sintaxis más consistente y poderosa que se alinea con las habilidades que los desarrolladores ya tienen con CSS. Esto hace que querySelector* sea generalmente preferido en el desarrollo moderno, especialmente cuando se necesitan selectores complejos.

    
// Ejemplos de selectores complejos con querySelector
// Seleccionar el tercer li dentro de un ul con clase "menu"
const item = document.querySelector("ul.menu li:nth-child(3)");

// Seleccionar elementos con atributos específicos
const inputs = document.querySelectorAll("input[type='text'][data-valid='true']");

    Fuente adicional: CSS-Tricks - DOM

  6. ¿Qué es un evento?

    Un evento es una acción o ocurrencia que sucede en el navegador o que es iniciada por el usuario, como hacer clic en un botón, cargar una página, mover el ratón o presionar una tecla. Los eventos son la base de la interactividad en JavaScript, permitiendo crear sitios web dinámicos que responden a las acciones del usuario.

    
// Ejemplo de evento de clic
document.getElementById("miBoton").addEventListener("click", function() {
    alert("¡Has hecho clic en el botón!");
});

    Fuente: MDN Web Docs - Referencia de eventos

    JavaScript.info añade que los eventos siguen un flujo de tres fases: captura (desde la raíz al elemento objetivo), objetivo (el elemento donde ocurrió el evento) y burbujeo (desde el elemento objetivo hacia la raíz). Este modelo de propagación permite implementar patrones como la delegación de eventos, donde un solo listener en un contenedor puede manejar eventos de múltiples elementos hijos.

    
// Ejemplo de delegación de eventos
document.getElementById("lista").addEventListener("click", function(event) {
    if (event.target.tagName === "LI") {
        alert("Has hecho clic en un elemento de la lista: " + event.target.textContent);
    }
});

    Fuente adicional: JavaScript.info - Bubbling and Capturing

  7. ¿Qué es un listener?

    Un listener (o escuchador de eventos) es una función que "escucha" y responde cuando ocurre un evento específico. Se registra para un elemento del DOM y un tipo de evento concreto, y se ejecuta automáticamente cuando ese evento ocurre.

    
// Agregar un listener de evento
elemento.addEventListener("click", funcionManejadora);

// Ejemplo con función anónima
document.getElementById("miEnlace").addEventListener("mouseover", function(event) {
    console.log("El mouse está sobre el enlace");
    event.preventDefault(); // Prevenir comportamiento predeterminado
});

// Eliminar un listener
elemento.removeEventListener("click", funcionManejadora);

    Fuente: MDN Web Docs - addEventListener

  8. ¿Qué es un callback?

    Un callback es una función que se pasa como argumento a otra función y se ejecuta después de que se complete alguna operación o evento. En el contexto del DOM, los callbacks se utilizan comúnmente como manejadores de eventos, ejecutándose cuando ocurre un evento específico.

    
// Ejemplo de callback como manejador de eventos
function saludar(nombre) {
    alert("Hola, " + nombre);
}

document.getElementById("boton").addEventListener("click", function() {
    saludar("Usuario"); // saludar es llamada como callback después del clic
});

// Otro ejemplo común: setTimeout con callback
setTimeout(function() {
    console.log("Este mensaje aparece después de 2 segundos");
}, 2000);

    Fuente: MDN Web Docs - Función de callback

  9. ¿Qué es un objeto de evento?

    Un objeto de evento es un objeto que se crea automáticamente cuando ocurre un evento y se pasa como parámetro a la función manejadora del evento. Contiene información sobre el evento que ocurrió, como el tipo de evento, el elemento objetivo, coordenadas del mouse, teclas presionadas, etc.

    
// Accediendo a información del objeto evento
document.addEventListener("click", function(event) {
    console.log("Tipo de evento: " + event.type);
    console.log("Elemento objetivo: ", event.target);
    console.log("Coordenadas: " + event.clientX + ", " + event.clientY);
    
    // Detener la propagación del evento
    event.stopPropagation();
    
    // Prevenir el comportamiento predeterminado
    event.preventDefault();
});

    Fuente: MDN Web Docs - Event

    Según la documentación de Google Developers sobre rendimiento web, es importante manejar los objetos de evento de manera eficiente. Cuando trabajamos con eventos que pueden dispararse frecuentemente (como mousemove o scroll), debemos implementar técnicas de "throttling" o "debouncing" para limitar la frecuencia con que procesamos estos eventos y evitar problemas de rendimiento.

    
// Ejemplo de debouncing para eventos frecuentes
function debounce(func, wait) {
    let timeout;
    return function() {
        const context = this;
        const args = arguments;
        clearTimeout(timeout);
        timeout = setTimeout(() => func.apply(context, args), wait);
    };
}

// Usar el debounce con un evento de scroll
window.addEventListener("scroll", debounce(function(e) {
    console.log("Scroll procesado una vez después de detenerse");
}, 150));

    Fuente adicional: Google Developers - Debounce Your Input Handlers

Conclusiones

Experiencia general

En general, esta actividad me hizo ver las increíbles capacidades que tiene la IA. Sobre todo me impresionó su capacidad para conectar varios archivos HTML y css, y manipularlos de la mejor manera posible. Sin embargo, considero que en muchas ocasiones se confundía porque le pedía varias cosas, y se le olvidaba responder las preguntas que le hacía.

¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

Usaría ChatGPT para tareas sobre todo de texto y análisis de preguntas y/o búsquedas en internet. Por otro lado, considero que Copilot (Claude específicamente) es más útil para tareas de programación, ya que procesa instrucciones más complejas y, en el caso del uso en VisualStudio, me permite ver varios archivos y analizarlos a la vez.

Ventajas de usar AI para programar

Considero que la ventaja de la IA son que, en primer lugar, ayuda a tener un mayor entendimiento de un tema cuando se analizan sus respuestas. Además, sirve mucho para encontrar errores y soluciones a éstos de manera más rápida. Finalmente, considero que también ayuda mucho a realizar tareas más complejas y/o repetitivas.

Desventajas de usar AI para programar

Las desventajas de la IA son que muchas veces pueden hacernos más flojos y que desfavorecen el aprendizaje si no se usa correctamente. Por otro lado, la IA no siempre tiene la respuesta correcta y puede confundirnos.
