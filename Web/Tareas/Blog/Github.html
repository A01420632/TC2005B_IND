Github

En esta sección, se tratará el flujo de trabajo de Github y Git, así como los comandos de Git que se usan.

Flujo de trabajo

  1. Crear un repositorio
    1. Ir a Github.com
    2. Hacer clic en "New repository"
    3. Configurar el repositorio
  2. Clonar el repositorio
    1. Ir a la sección de code
    2. Copiar la URL del repositorio o la SSH Key
    3. Ir a Git Bash
    4. Ir al directorio en donde se quiere copiar el repositorio
    5. Usar el comando git clone + "link/clave SSH"
  3. Entrar al repositorio
    1. Ir a Git Bash
    2. Usar el comando cd+"link del directorio en el que está el respositorio"
    3. Si se quiere visualizar el contenido del repositorio, se puede usar el comando ls
    4. Para ver todo el contenido con detalles/descripción: ls -al
  4. Hacer cambios
    1. Si se quiere hacer cambios:
      • Crear archivo: touch "nombre.tipo"
      • Editar archivos: vi "archivo"
      • Copiar archivo: cp
      • Borrar archivo: rm
      • Mover archivo: mv>
    2. Si se quiere visualizar un archivo:
      • Mostrar 10 primeras líneas de un archivo: head
      • Mostrar 10 últimas líneas: tail
    3. Para más información, ir a: Comandos de terminal
  5. Git add, commit, push
      Una vez que se hicieron los cambios, se tienen que subir al repositorio en línea (Los hechos a través de Git Bash son solo en el directorio hasta que se haga un push)
    1. Para verificiar el status de nuestros cambios, usar el comando: git status
    2. Cuando se quiere preparar un archivo para subirlo al repo (staging area): git add + "nombre"
    3. Verificar si ya pasó a staging area: git status
    4. Posteriormente, se hace commit de los cambios que ya están terminados (se suben al repositorio local): git commit -m + "Descripción de cambios realizados"
    5. Verificar nuevamente el status para ver si se hizo el commit correctamente: git status
    6. Hacer un git push para actualizar los cambios en el repo: git push -u origin main

Comandos de Git

En esta sección se mostrarán los comandos de git a detalle:

  1. git status: Verifica el estado de los cambios
  2. git add: Se pasan los archivos al staging area
  3. git commit -m: Sube los cambios hechos al repositorio local
  4. git push -u origin: Sube los cambios hechos al repositorio en línea
  5. git pull origin: Se bajan los cambios del repositorio (branch main) en línea al local
  6. git pull origin "rama"Se bajan los cambios de la branch main a la rama
  7. git log: Visualizar el historial de commits
  8. git restore --staged: Regresa al directorio (unstage) al archivo
  9. git checkout -b "rama"Crea una rama y se traslada a ella
  10. git checkout "rama"Cambia de rama
  11. git switch: Misma función que la anterior
  12. git branch: Enlista las ramas existentes
  13. git branch -a: Enlista las ramas tanto locales como remotas
  14. git branch "rama": Crea una rama
  15. git merge "rama": Hace un merge de los cambios (de la rama mencionada a la rama en la que te encuentras)
  16. git branch -d "rama": Elimina la rama
  17. git branch -D "rama": Asegura que se borra la rama en caso de que fue un intento fallido y no importa que no se haya hecho merge
  18. git fetch prune--: Elimina el rastro que dejan las ramas después de ser eliminadas
  19. git push origin -d "rama": Después de eliminar una rama localmente, se hace un push para eliminarla del repositorio remoto
